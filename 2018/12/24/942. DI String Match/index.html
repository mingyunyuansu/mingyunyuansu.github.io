942.DI String Match

Given a string S that only contains “I” (increase) or “D” (decrease), let N = S.length.

Return any permutation A of [0, 1, …, N] such that for all i = 0, …, N-1:

  • If S[i] == “I”, then A[i] < A[i+1]
  • If S[i] == “D”, then A[i] > A[i+1]

挺简单的，想了几分钟，又写了几分钟就OK了。我的想法是，首先建一个0到N的有序数组作为初始的A，然后顺着过一遍S，根据碰到的’D’和’I’，判断对应位置的A[i]和A[i+1]是否符合要求，不符合就swap(A[i], A[i+1])。因为A比S多一位，所以没有边界问题。但是直觉就是肯定会出现这种情况：swap当前的数字影响了前面已经调整好了的数字。不过没关系，仔细一想就会发现这种情况只会出现在连续的’I’段或者连续的’D’段中，比如对于[0 1 2]和”DD”，对第一个’D’执行swap，得到[1 0 2]，对第二个’D’再执行swap得到[1 2 0]，这时就会发现第一个’D’受到了影响，变得不符合条件。那么很自然地想到，每次swap时确保在当前位置之前的连续的’D’段（或’I’段）都符合条件就行了，代码如下：
