伸展树(Splay Tree)

Jun 16, 2019 | 算法技术

在AVL树之后又介绍了一种树——伸展树(Splay Tree)。

伸展树保证从空树开始连续M次对树的操作最多花费O(M logN)时间。尽管不排除一次操作花费O(N)时间的可能，但是发生不良操作的概率很低。总的来说，这种树保证总体的操作时间令人满意。

具体来讲，伸展树的核心思想是，每当一个结点被访问后，就要经过一系列旋转使它被放到根上（跟程序访问的时间空间局部性有关，因为被访问过的数据可能很快还会被访问）。并且最关键的是，每一次这样的移动，我们要把树“伸展”，使得这棵树的深度尽量浅，结点分布均匀，最终同样达到平衡的效果。

至于如何实现？书上没有给代码，只给了讲解。首先想到，AVL树中的单旋转其实可以把结点向上提，所以可以仿照AVL的旋转方式不断把被访问的结点向上提到顶端。但是书上给出了一个例子表明，这样会把其他某个结点推到同样很深的位置，所以同样无法达到均衡深度的效果。因此正确的做法是：

  • 若是被访问结点跟其父节点，祖父节点的关系是AVL树中的左右、或右左的交叉形式（zig-zag型），则执行一次双旋
  • 若是左左，或右右的zig-zig型，则执行一次。。。不知叫啥的变换，类似镜像，但是父子关系完全颠倒

双旋的部分与AVL一样。
