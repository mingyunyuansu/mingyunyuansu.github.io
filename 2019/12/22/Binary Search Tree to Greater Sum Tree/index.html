Binary Search Tree to Greater Sum Tree

Dec 22, 2019 | 算法树

好久不见。

每到圣诞我的日子似乎就会变得可怜起来，今年大概是在图书馆孤独地刷leetcode吧。不得不承认，可能比起自己漫无目的地学习网络安全，我还是对算法这种亘古不变的东西感兴趣一点，它更接近数学，我果然更喜欢这样的知识。学校教的完全没用，我还不得不写一堆安全报告之类的狗屁内容，不知道为什么这帮人那么喜欢在engineering的课程里插那么多管理类东西，我很难受。不知道当初去美国，或者坚持去日本，甚至就二战或接受调剂会不会好点。但不管怎样，一句来都来了解决一切心态不平衡。唯一学到的教训就是任何时候都不要放开自己人生的舵，不要让别人来掌握你的人生。这真是所谓因为年轻所犯下的错误啊。

废话讲完，直接看题吧。

Given the root of a binary search tree with distinct values, modify it so that every node has a new value equal to the sum of the values of the original tree that are greater than or equal to node.val.

As a reminder, a binary search tree is a tree that satisfies these constraints:

The left subtree of a node contains only nodes with keys less than the node’s key.

The right subtree of a node contains only nodes with keys greater than the node’s key.

Both the left and right subtrees must also be binary search trees.

我认为这是一道非常好的题，出现在尚可的公司的面试题中不奇怪。同时涉及了递归和树搜索两个我的一生之敌。有时真的怀疑自己的思维模式不适合学编程，反正递归和树遍历我现在也不能顺畅地思考。

我没有独立做出这道题来。很容易就能看出一个节点的值其实就是所有“在他右边的节点”的值之和。但是我不知道怎么把这个逻辑归纳为计算机语言，于是直接看别人代码了。

需要注意两行递归的使用。第一句递归没什么好说的，直接往右深入的标准做法。但是第二句递归比较巧妙，在这个位置递归的话就能带着上面得到的sum进行左深入，然后把左边的累加值也算出来（因为左节点的值比其父节点还要大，这里带着sum递归天衣无缝）。

同时注意一下函数间传值求和的做法。这里写的很干练，值得借鉴，递归间进行值传递经常把我绕进去，最后不得不用全局变量来进行求和，这也是我头脑依然不适应递归的证据。
