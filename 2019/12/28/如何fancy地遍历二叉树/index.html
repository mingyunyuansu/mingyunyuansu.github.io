如何fancy地遍历二叉树————Morris Traversal

Dec 28, 2019 | 技术

树这种数据结构的遍历一直是我的弱项。这跟我大二上的数据结构完全没有上（完全在打刀塔）有关，也跟实在是不适应递归有关。不管怎么说，当年课堂上教地很难很深，但是最后遍历二叉树大家都只会用递归，耍点花招的话就用迭代+栈，其实二者一模一样。我一直以为我差不多补上这块了，直到我遇上了这一题：

99. Recover Binary Search Tree

这也是我第一次挑战leetcode上的hard难度的题。

题目很简练（所谓题目越短越难），就是给定一个二叉搜索树BFS，但是其中有两个结点的位置被调换了导致BFS失效，要求把这个搜索二叉树恢复。

看见题目就有思路了（其实很多时候只求答案的话往往都能找到简单的解法）。关键在于标准的中序遍历也就是in-order traversal刚好可以把一棵搜索二叉树按严格的递增顺序遍历。我们只需要对题给的BFS跑一遍in-order traversal就能得到当前搜索二叉树的“顺序”，可以当做一个有序数列。再仔细思考一下，这其实相当于一个递增数列里，有两个值交换了位置，那这两个错误值出现时的特征是很明显的：那个位置的数比其前一位的数小，违反了递增规则。只要用一个if语句检测这个条件即可。再细化一下会发现，这个条件出现的时候，左边的错误位是“前一个”，右边的错误位是其本身。

这么说太抽象了，举个例子：

但是hard题显然不会这么轻易放过你，follow up里直接告诉你空间复杂度O(n)的解法太naive（递归的本质是栈的叠加，递归的空间复杂度就是树的最大高度O(n)），敢不敢换一个？于是就出现了奇跡のMorris Traversal！关于Morris Traversal的讲解请直接移步北大小姐姐的无敌笔记。因为我也第一次接触，而且她做的超详细超正规，跟中国网路上一水的抄袭和半吊子解释完全不一样，我就不献丑了。

我的理解是，之所以需要靠递归或者栈来遍历树是因为树没有回退的办法，遍历是单向的只能往下不能往上。这就造成遍历完一个分支后无法确定下一个分支的起点。而Morris Traversal恰好解决了这个问题：在保留当前位置cur之后，提前下潜(prev)之后确认当前分支中最后一个遍历的叶子结点，然后将其手工连回分支起点，这样就形成了一个环的通路。等到真的遍历过“最后一枚”之后，再把手动添加的辅助线删去。中序和前序都是一样的思路。

空间复杂度自不用说是常数级，多了cur和一个下潜用的prev而已（其实我觉得她起的prev不好，先不管了）；时间复杂度她说也是O(n)，因为n个结点的二叉树有n-1条边，而每条边最多遍历两次，因此是O(n)级而不是O(logn*n)。嘛，我历来不会算这个，她说是就是吧。这么来看Morris Traversal完爆递归写法，应该多写几次背下来以增加面试筹码。

后序遍历似乎很复杂，我没看，但是后序遍历本身也很少用到，暂时就跳过吧。她自己查找之后发现后序遍历资料很少，但还是愣是被她钻研出来了，这就是对比，我服。她的步骤和说明都是非常详细的那种，单画那几个步骤图就要好半天吧，可能这就是北大吧，跟我这种懒得钻太深也懒得作图搞格式的人完全不一样。
